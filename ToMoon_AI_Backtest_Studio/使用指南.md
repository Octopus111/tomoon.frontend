## 🎯 快速开始

### 1. 启动后端服务
```bash
cd ToMoon_AI_Backtest_Studio/backend
python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 访问应用
在浏览器打开：**http://localhost:8000**

⚠️ **重要**：不要双击 `index.html` 文件！必须通过 `http://localhost:8000` 访问。

## 🔑 AI 配置

### 配置步骤
1. 点击右上角 **⚙️ AI 偏好设置**
2. 填写以下信息：
   - **Base URL**: `https://dashscope-intl.aliyuncs.com/compatible-mode/v1`（默认）
   - **API Key**: 你的阿里云 DashScope API Key（格式：`sk-...`）
   - **Model**: `qwen-plus` 或 `qwen-max`
3. 点击 **💾 保存设置**
4. 点击 **🔌 测试 AI 连接** 验证配置

### 获取 API Key
1. 访问 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
2. 注册/登录账号
3. 创建 API Key
4. 确保账户有足够余额

### 测试连接
- ✅ **成功**: 显示 "AI Connected | Ready"，并生成测试策略
- ❌ **失败 (401)**: API Key 错误，检查是否复制正确（无空格）
- ❌ **失败 (403)**: 模型未开通或权限不足
- ❌ **失败 (404)**: Base URL 错误

## 💬 渐进式策略构建

### 核心理念
**边聊天边构建** - 无需一次性说完所有条件，可以逐步补充完善。

### 示例对话流程

#### 第一轮：定义入场信号
```
你：SMA20 上穿 EMA50 做多
AI: ✅ 已更新策略:
    - 新增入场条件：SMA20 上穿 EMA50
    👌 已收到。你可以继续补充：入场条件、离场条件、止盈止损、最大持仓。
    💡 示例：'止盈 3%' 或 'RSI 低于 40 入场' 或 '最大持仓 100 根 K 线'
```

#### 第二轮：添加过滤条件
```
你：RSI 低于 30 才入场
AI: ✅ 已更新策略:
    - 新增入场过滤：RSI14 < 30
    👌 已收到。你可以继续补充：入场条件、离场条件、止盈止损、最大持仓。
```

#### 第三轮：设置止盈止损
```
你：止盈 3%，止损 1.5%
AI: ✅ 已更新策略:
    - 更新止盈：3.00%
    - 更新止损：1.50%
    🎯 策略已基本完整！你可以：1) 继续微调；2) 点击「运行回测」按钮测试真实表现。
```

#### 第四轮（可选）：生成完整代码
```
你：生成完整策略代码
AI: 正在调用 AI 生成完整策略代码...
    ✅ AI 策略已生成:
    - LLM Successfully Generated Strategy
    - 已按聊天内容生成可执行策略代码
    - 支持真实市场数据回测并返回可审计结果
    💡 现在可以点击「运行回测」按钮开始真实回测，或继续微调策略。
```

### 支持的指令

#### 入场条件
- `SMA20 上穿 EMA50 做多`
- `RSI 低于 30 入场`
- `RSI 高于 70 做空`
- `MACD 金叉入场`

#### 离场条件
- `SMA20 下穿 EMA50 平仓`
- `RSI 高于 80 离场`

#### 风险管理
- `止盈 3%`
- `止损 1.5%`
- `最大持仓 100 根 K 线`
- `最多持仓 50 bar`

#### 逻辑修改
- `入场条件满足其一就行` → OR 逻辑
- `入场条件需要同时满足` → AND 逻辑
- `离场条件满足其一就平仓` → OR 逻辑

#### 特殊指令
- `清空策略` / `重置` → 清空所有条件
- `生成代码` / `生成完整策略` → 调用 AI 生成可执行代码
- `运行回测` → 开始真实回测

## 📊 运行回测

### 步骤
1. **构建策略**: 通过对话至少定义一个入场条件
2. **点击运行**: 点击右上角 **▶️ 运行回测**
3. **配置参数**:
   - **Market**: XAUUSD, EURUSD, BTCUSD, CME_ES
   - **Resolution**: 1m, 5m, 15m, 1H
   - **日期范围**: From / To
   - **初始资金**: 默认 $100,000
   - **手续费**: 默认 2.5
   - **点差**: 默认 0.0002
   - **滑点**: 默认 0.0005
4. **确认开始**: 点击 **开始真实回测**

### 回测结果
回测完成后会显示：
- **Net Profit**: 净利润
- **Win Rate**: 胜率
- **Max DD**: 最大回撤
- **Trades**: 交易次数
- **资金曲线图**: 账户权益变化
- **K 线图**: 价格走势与交易标记

1. 在画布上点击任意指标节点
2. 弹出微调聊天窗口
3. 输入调整指令，例如：
   - `把 RSI 过滤从 30 调整到 45，并解释原因`

- **Live Strategy JSON**: 实时显示当前策略 JSON
- **Generated Strategy Code**: 显示 AI 生成的可执行代码

## ❓ 常见问题

### Q1: 为什么显示 "模型服务不可用"？
**A**: 可能原因：
1. 未配置 API Key → 在 AI 偏好设置中配置
2. API Key 错误 → 点击测试连接验证
3. 网络问题 → 检查网络连接
4. 余额不足 → 检查阿里云账户余额

即使 AI 不可用，仍可使用**本地解析模式**构建策略！

### Q2: API Key 错误 (HTTP 401) 怎么办？
**A**: 检查以下几点：
1. API Key 是否正确复制（无多余空格）
2. API Key 格式是否为 `sk-...`
3. API Key 是否已在阿里云控制台激活
4. 账户余额是否充足
5. Base URL 是否正确

### Q3: 如何切换回本地解析模式？
**A**: 两种方法：
1. 不配置 API Key，自动使用本地解析
2. AI 调用失败时，自动回退到本地解析

### Q4: 回测结果为 0 交易怎么办？
**A**: 可能原因：
1. 条件过于严格（如 SMA 上穿 + RSI 低于 30 同时满足）
2. 日期范围内确实没有符合条件的信号
3. 时间周期太短

建议：
- 放宽条件（如 RSI 低于 40 改为 50）
- 延长回测时间范围
- 改用 OR 逻辑（满足其一即可）

### Q5: 如何查看详细的回测报告？
**A**: 回测完成后：
1. 结果弹窗显示核心指标
2. 查看资金曲线图了解走势
3. 查看 K 线图了解交易点位
4. 查看 Live Strategy JSON 了解策略细节
